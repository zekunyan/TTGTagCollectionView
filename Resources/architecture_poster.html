Swift-first tag layout framework

TTGTagCollectionView Architecture

A UIKit tag engine that separates data, layout, and rendering. It keeps Objective-C call sites compatible while the core layout and text tag stack are now Swift-based and cache-aware.

2 APIsTextTagCollectionView for styled text, TagCollectionView for any custom UIView.
6 modesLeft, center, right, fill space, fill width, and fill width except last line.
Fast cellsPrecompute content size and reuse row heights for dense UITableView lists.
OC readySwift internals with @objc names and generated TTGTags-Swift.h imports.

Render Layer

Text tags render through lightweight component views. Custom tags are regular UIViews supplied by the app.

Performance Strategy

MeasureCache text tag sizes instead of repeatedly asking UILabel to size every tag.
LayoutCache layout outputs for repeated size arrays and view configuration.
UITableViewUse contentSize(for:width:...) to precompute row height before cell display.
ReloadBatch tag configuration and reload once per cell reuse pass.

Why It Works

  • ✓Layout is side-effect-free, testable, and easy to cache.
  • ✓TextTagCollectionView hides rendering details behind a small data model.
  • ✓Auto Layout sees stable intrinsicContentSize and system fitting sizes.
  • ✓Objective-C apps keep existing selectors while using the Swift implementation.
Use TextTagCollectionViewBest for styled text chips, selected states, rich text, gradients, and accessibility.
Use TagCollectionViewBest for avatars, buttons, custom controls, or any UIView-backed tag content.
Use precomputed sizeBest for dense tags inside self-sizing UITableViewCell or UICollectionView layouts.
